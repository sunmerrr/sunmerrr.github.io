### 게임 세팅 전체 UI 및 로직 변경
**내용**: 게임 생성이 조금 더 적은 뎁스를 거쳐서 될 수 있도록 하기위해 리뉴얼 함 + 하는김에 리팩토링까지    
**스택**: React-Hook-Form, zustand     
**주요 수정 사항**:     

1. 게임 생성 개선
1. 역할 분담: zustand 와 useContext, React-Hook-Form 의 역할 분담이 제대로 되어있지 않은 부분 리팩토링
1. 코드 흐름 개선: useEffect가 남발되어서 코드 흐름이 제대로 읽히 않던 부분 리팩토링

#### 1. 게임 생성 개선
- 요약: 개임 생성의 `create - prepare - update - finalize` 순서를 create 한 단계로 변경    
기존에 토너먼트 게임의 경우는 게임이 `생성 - 준비 - (수정) - 배포요청 - 배포` 순으로 작업되어 있었다.    
히스토리를 가볍게 말하자면 이렇다.    
복잡한 게임 생성의 허들을 낮추기 위해 꼭 필요한 것들만 첫번째 뎁스에 노출 -> 두번째 뎁스를 반드시 지나며 해당 부분에서 배포 가능 -> 여기까지는 `생성 - 준비 - (수정) - 배포` 순서로 짧으면 2 뎁스 안에 게임을 만듬 -> Web3가 포함되면서 바로 배포될 수 없고 `배포요청` 단계를 추가하게 됨 -> 리팩토링 전의 괴물과 같은 모습으로 변함
